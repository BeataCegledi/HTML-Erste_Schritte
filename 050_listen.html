Listen in HTML

In diesem Abschnitt kann ich alles über Listen in HTML nachlesen. HTML kennt:

<ol> - Geordnete /Sortierte Liste (ordered list) - Nummerierung oder Aufzählungszeichen
<ul> - Ungeordnete / unsortierte Liste (unordered list) - Aufzählungszeichen
<dl> - Definitionsliste / beschreibende Liste (description list) - Definitionen, Begriffe und Beschreibungen

1. Wann nutzen wir welche Liste?

  • <ol> → Wenn die Reihenfolge wichtig ist (z.B. Anleitung, Rangliste, Rezept, Top 10 Listen, etc.)
  • <ul> → Wenn die Reihenfolge egal ist (z.B. Einkaufsliste, Liste von Features, etc.)
  • <dl> → Für Begriffe mit Erklärungen, Definitionen oder Beschreibungen (z.B. Glossar, FAQ, Wörterbuch, etc.)

2. Sortierte Listen (<ol>)

Eine Sortierte Liste nummeriert die Listenelemente automatisch.

Beispiel (Inhaltsverzeichnis):

  1. Grundaufbau eines Links
  2. Das Attribut href
  3. Link in einem neuen Tag öffnen
  4. Email und Telefon Links
  5. Interne Links (Sprungmarker)

Wichtige Attribute:

  • type="1": 1, A, a, I, i -> Bestimmt das Nummerierungsverfahren
  • start: Startzahl für die Nummerierung
  • reversed: Nummerierung rückwärts

Beispiel (Tiere):

  1. Hund
  2. Katze
  3. Kuh
  4. Meerschweinchen
  5. Pinguin
  1. Hund
  2. Katze
  3. Kuh
  4. Meerschweinchen
  5. Pinguin
  1. Hund
  2. Katze
  3. Kuh
  4. Meerschweinchen
  5. Pinguin

3. Unsortierte Listen (<ul>)

Eine Unsortierte Liste stellt Aufzählungen dar, bei denen die Reihenfolge /Priorität egal ist.

Beispiel (Städte):

  • München
  • Rom
  • Nürnberg
  • Budapest

Wichtige Attribute:

  • type: none, circle, square, disc (Standard) -> Art der Aufzählungszeichen

Beispiel (Städte):

Circle:

  • München
  • Rom
  • Nürnberg
  • Budapest
  • Bangkok

Square:

  • München
  • Rom
  • Nürnberg
  • Budapest
  • Bangkok

Disc:

  • München
  • Rom
  • Nürnberg
  • Budapest
  • Bangkok

None:

  • München
  • Rom
  • Nürnberg
  • Budapest
  • Bangkok

4. Beschreibende Listen (<dl> = Description List)

Eine Beschreibende Liste stellt Begriffe mit ihren Erklärungen oder Definitionen dar.
Sie besteht aus <dt> (Definition Term) für den Begriff
und <dd> (Definition Description) für die Beschreibung oder Erklärung.

Beispiel (Begriffe aus der IT):

<dt>HTML</dt>
<dd>HyperText Markup Language - Aufzeichnungssprache zur strukturierung von Webseiten</dd>
CSS
Cascading Style Sheets - Formatierungssprache für das Design und Layout von Webseiten
JavaScript
Programmiersprache für interaktive Webseiten und Webanwendungen
HTTP
HyperText Transfer Protocol - Protokoll für die Übertragung von Daten im Web
PHP
Personal Home Page - Scriptsprache für die Entwicklung dynamischer Webseiten
API
Application Programming Interface - Schnittstelle für die Interaktion zwischen Softwareanwendungen

Mehrere Beschreibungen für einen Begriff

<dt>HTML</dt>
<dd>HyperText Markup Language - Aufzeichnungssprache zur strukturierung von Webseiten</dd>
<dd>Wird verwendet, um die Struktur und den Inhalt von Webseiten zu definieren</dd>
<dd>Besteht aus Elementen, die durch Tags dargestellt werden (z.B. <h1>, <p>, <a>, etc.)</dd>
CSS
Cascading Style Sheets - Formatierungssprache für das Design und Layout von Webseiten
Ermöglicht die Trennung von Inhalt (HTML) und Design (CSS)
JavaScript
Programmiersprache für interaktive Webseiten und Webanwendungen
Wird verwendet, um dynamische Inhalte zu erstellen und Benutzerinteraktionen zu ermöglichen

5. Verschachtelte Listen

Listen können auch in anderen Listen eingebettet werden, um hierarchische Strukturen darzustellen. (z.B. Menüs und Kategorien, etc.)

Beispiel (Elektrogeräte):

  • Produkte
    1. Handys
      • iOS
      • Androids
    2. Laptops
    3. Tablets
  • Service
    1. Reparatur
    2. Garantie
    3. Kundendienst

Anleitung: Verschachtelte Listen in HTML

  1. Erstelle zuerst eine normale Liste mit <ul> oder <ol>
  2. Lege einen Listeneintrag mit <li> an
  3. Schreibe den Text für den Haupteintrag in dieses <li>
  4. Öffne innerhalb desselben <li> eine neue Liste (<ul> oder <ol>)
  5. Lege in dieser neuen Liste die Unterpunkte mit <li> an
  6. Schließe zuerst die Unterliste und danach das äußere <li>
