CSS einbinden - Inline, Intern und Extern

Es gibt drei Arten, CSS in HTML eiinzubinden

  1. Inline CSS = direkt im HTML geschrieben
  2. Interne CSS = im <style>-Tag im head geschrieben
  3. Externe CSS = in einer .css Datei angelegt

1. Inline CSS

Inline CSS wird direkt im HTML Tag mit dem Attribut style geschrieben!

Dieser Text ist mit Inline CSS gestylt. 

                    <p style="color:blueviolet;              -> color = Farbe der Schrift
                                background-color:aquamarine; -> background-color = Hintergrundfarbe
                                padding: 8px;">              -> padding = Innenabstand (Platz zwischen 
                                                                        dem Imhalt eines Elements und seinem Rand)
                
                                Farben können in englischen Namen, hex-Werten oder aber RGB dargestellt werden!

Wann ist das sinnvill? Fast nie, nur für schnelle Test oder einzelne Ausnahmen.
Warum ist das schlecht? Unübersichtlich und sehr schlecht wartbar.

2. Interne CSS wird im head mit <style>-Tag geschrieben

Interne CSS steht im head im <style>-Tag der HTML Datei und es gilt für die gesamte Seite!

Dieser Bereich ist mit internem CSS gestylt. 

                    lt;style>
                        Selektor{
                            eigenschaft: wert;
                        }

                        body{
                            font-family: Arial, Helvetica, sans-serif;  -> bestimmt die Schritfart
                            line-height: 1.5;                           -> bestimmt den Zeilenabstand
                        }

                        .intern{ /*spricht die Klasse intern mit . und Klassenname an*/
                            color:darkgreen;               -> color = Farbe der Schrift
                            background-color: #e6ffe6;     -> background-color = ist die Hintergrundfarbe
                            padding: 4px;                  -> Innenabstand
                        }

                        .box{
                            border: 5px black solid;    -> border ist der Rahmen | 2px ist die Rahmengröße 
                                    | black ist die Rahmenfarbe | Solid ist die feste/ darstellund des Rahmens
                            padding: 10px;              -> Innenabstend
                            margin:  10px 0;            -> margin = Außenabstand (oben, rechts, unten links) 
                                                | 10px steht für oben + unten und 0px steht für rechts + links*/
                        }

Wann ist es sinnvoll? Bei kleineren Projekten und Übungen.
Warum ist es sinnvoll? Alles in einer Datei ist und es damit leichter zu verstehen ist.

3. Externe CSS (separate Datei)

Externe CSS liegt in einer eigenen Datei (z.B.: styles.css) und wird mit einem <link>-Tag eingebunden.

Dieser Bereich ist mir einem externem CSS gestylt wurden. 

                    .extern{
                    color:antiquewhite;         -> Farbe der Schrift
                    background-color: #3333cc;  -> Hintergrundfarbe
                    padding: 10px               -> Innenabstand
                    }

Wann ist es Sinnvoll? Fast immer bei "echten" Webseiten!
Warum?Sauber, wiederverwendbar und leicht wartbar.

4. Priorität und Reihenfolge

Wenn mehrere Styles auf das selbe Element wirken, gilt: (wenn ich gut verstanden habe die Reihenfolge des schreiben im head ist auch entscheidend)

  1. Inline CSS (höchste Priorität)
  2. Interne Css (mittlere Priorität)
  3. Externes CSS (niedrigste Priorität)
