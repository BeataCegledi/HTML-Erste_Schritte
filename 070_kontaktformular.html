Kontaktformulare in HTML

Formulare helfen uns dabei, Daten von Nutzern einzusammeln.
Die wichtigste Bausteine sind:

  • <form> = der Formular Rahmen (Container)
  • <label> = Beschrifung für die Eingabefelder
  • <input> = Eingabefeld (z.B.: Name, Passwort, E-mail, Telefonnummer, etc.)
  • <textarea> = Ein großes Textfeld (z.B.: für Nachrichten)
  • <select> und <option> = Dropdown Auswahl
  • <button> = Button (z.B.: Absenden, Zurücksetzen)

1. Das einfache Kontaktformular

Ein Kontakformular besteht meistens aus Name, E-Mail und Nachricht

for (bei <label>) + id
Verbindet Label und Feld. Klickst du auf das Label, springt der Cursor ins Feld

2. Die wichtigsten Form-Attribute: action und method

action=""
Wohin die Daten geschickt werden (Bsp.: Server Adresse oder ein Skript)
method="GET"
Daten werden in die URL (=Uniform Resource Locator) geschrieben.
Gut für eine Suche, schlecht für sensiblen Daten.
method="POST"
Daten werden im Hintergrund gesendet (Standard Formular).

HINWEIS: Ohne echten Server passiert beim Absenden oft nichts Sichtbares
aber HTML "baut" das Formular korrekt!

3. Input-Typen und ihre Funktionen

Der Typ sagt dem Browser, welche Art von Daten erwartet werden. Dadurch bekommt man die passenden Tastaturen am Handy/Tablet und manchmal auch eine automatische Überprüfung.

4. Pflichtfelder und einfache Validierung

Mit required kann man Felder Verpflichtend stellen.
Der Browser verhindert dann das Absenden, wenn etwas fehlt.

HTML Validierung
... prüft Eingabe direkt im Browser, bevor das Formular abgeschickt wird.
Vorteil: Sofortige Rückmeldung an den Benutzer, falls etwas falsch eingegen wurde.
Nachteil: Kann mit F12 oder eigenen Anfragen umgangen werden.
Server Validierung
...prüft die Daten nach dem Absenden noch einmal auf dem Server.
Vorteile: Kann nicht mit F12 umgangen werden | Sorgt für Sicherheit
Nachteil: Nicht so einfach wie HTML Validierung in überprüfung und Geschwindigkeit
